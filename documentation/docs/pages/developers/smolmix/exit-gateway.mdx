# Target a Specific Exit Gateway

import { Callout } from 'nextra/components'

By default, `Tunnel::new()` auto-selects an IPR (Internet Packet Router) exit gateway. This guide shows how to target a specific one.

## Why target a specific gateway?

- **Geographic preference** — route traffic through an exit in a specific region
- **Reliability** — stick to a gateway you've tested and trust
- **Debugging** — isolate issues to a specific exit node
- **Testing** — verify your application works against different exits

## Using TunnelBuilder

Pass a `Recipient` address to the builder:

```rust
use smolmix::{Tunnel, Recipient};

let ipr: Recipient = "gateway-nym-address...".parse()?;
let tunnel = Tunnel::builder()
    .ipr_address(ipr)
    .build()
    .await?;
```

## Accepting an address from the command line

The tutorials use this pattern to accept `--ipr` as a CLI flag:

```rust
let args: Vec<String> = std::env::args().collect();
let ipr_addr = args
    .iter()
    .position(|a| a == "--ipr")
    .and_then(|i| args.get(i + 1));

let mut builder = Tunnel::builder();
if let Some(addr) = ipr_addr {
    builder = builder.ipr_address(addr.parse().expect("invalid IPR address"));
}
let tunnel = builder.build().await?;
```

```sh
RUST_LOG=info cargo run -- --ipr <IPR_ADDRESS>
```

## Finding available IPR gateways

IPR gateways are exit gateways that have the IP Packet Router service enabled. You can find them through:

- **Nym API** — query the network explorer or API for gateways with IPR capability
- **Logs** — when `Tunnel::new()` auto-selects, it logs the chosen gateway at `info` level

<Callout type="info">
If the specified gateway is unavailable or doesn't support IPR, `Tunnel::builder().build()` will return an error. Fall back to auto-selection by omitting the `ipr_address()` call.
</Callout>

## Advanced: bring your own connection

If you've already established an `IpMixStream` (e.g. with custom gateway selection logic), you can bypass the builder entirely:

```rust
let tunnel = Tunnel::from_stream(my_ip_mix_stream).await?;
```

This is useful when you need full control over the mixnet client configuration or want to reuse an existing connection.
