# Nyx Blockchain

Nyx is a Cosmos SDK blockchain that provides coordination services for the Nym Network.

<Callout type="info">
To interact with the chain, see [Interacting with Nyx](../../developers/chain). To run a Validator, see the [Operator Documentation](../../operators/nodes/validator-setup).
</Callout>

## Role in the Network

The blockchain plays a supporting but fundamental role:

- **Topology registry**: Maintains the list of active nodes and their public keys
- **Token management**: The NYM token is a native token of the chain
- **Smart contracts**: Hosts contracts for mixnet coordination and credentials
- **Reward distribution**: Calculates and distributes node operator rewards

## Validators

Nyx Validators secure the blockchain via Proof-of-Stake consensus. They:

- Run the `nyxd` binary ([source code](https://github.com/nymtech/nyxd))
- Maintain a CosmWasm and IBC-enabled blockchain
- Process transactions and execute smart contracts

The validator set also includes nodes running the Nym API for credential issuance.

## Nym API

<Callout type="info">
For setup instructions, see the [Nym API Operator Guide](../../operators/nodes/validator-setup/nym-api).
</Callout>

A subset of Nyx validators operate **Nym API** instances, forming the "Quorum". These perform:

### Network Monitoring

- Send test packets through the mixnet
- Calculate routing scores for nodes
- Determine node reliability metrics

### Credential Issuance

- Generate partial blind signatures for zk-nyms
- Verify payment deposits
- Maintain double-spend protection (Bloom filter)

The Quorum uses threshold cryptography: no single member can issue credentials alone, and the system remains functional even if some members are offline.

## Smart Contracts

The Nyx chain is [CosmWasm](https://cosmwasm.com/)-enabled, hosting several key contracts:

### Mixnet Contract

The core contract for network coordination:

- Stores bonded node information (and removes on unbonding)
- Provides network topology for client routing
- Tracks delegation and bond amounts
- Manages reward calculations

Having the topology on-chain prevents attacks that plague P2P directory systems.

Code: [`contracts/mixnet`](https://github.com/nymtech/nym/tree/master/contracts/mixnet)

### Vesting Contract

Manages NYM token vesting:

- Creates and tracks vesting accounts
- Allows delegation of unvested tokens
- Enables participation in staking before full vest

Code: [`contracts/vesting`](https://github.com/nymtech/nym/tree/master/contracts/vesting)

### zk-nym Contract

<Callout type="info">
In the codebase, this contract is named `ecash` for historical reasons.
</Callout>

Hub for credential-related actions:

- Tracks NYM deposits for credential generation
- Manages blacklist for double-spend attempts
- Handles Gateway redemption of spent credentials

Called by:
- Payment backend (deposits)
- Gateways (redemption proposals)
- Quorum multisig (blacklist management)

### Multisig Contract

The Quorum uses a multisig contract (based on `cw3-flex-multisig`) to:

- Execute zk-nym contract actions
- Trigger reward payouts
- Manage blacklist proposals

## Contract Addresses
