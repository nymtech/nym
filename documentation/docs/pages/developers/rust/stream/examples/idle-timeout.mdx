# Idle Timeout

> Source: [`examples/stream_idle_timeout.rs`](https://github.com/nymtech/nym/blob/develop/sdk/rust/nym-sdk/examples/stream_idle_timeout.rs)

This example demonstrates how streams are automatically cleaned up after a period of inactivity. This prevents leaked entries when the remote side drops without the local side knowing.

## What this example shows

- Configuring a custom idle timeout via `MixnetClientBuilder`
- Observing EOF on a stream after the timeout elapses
- How the router's periodic cleanup works

## Configuring the timeout

The default idle timeout is 30 minutes. For this example, we set it to 2 seconds to observe the cleanup quickly:

```rust
let mut client = mixnet::MixnetClientBuilder::new_ephemeral()
    .with_stream_idle_timeout(Duration::from_secs(2))
    .build()
    .unwrap()
    .connect_to_mixnet()
    .await
    .unwrap();
```

## Open a stream to self

The example opens a stream to its own address — useful for testing without a second process:

```rust
let our_address = *client.nym_address();
let mut listener = client.listener().unwrap();
let mut outbound = client.open_stream(our_address, None).await.unwrap();
let mut inbound = listener.accept().await.unwrap();
```

## Use the stream, then go idle

After a successful write/read cycle, both sides stop using the stream. The router's background cleanup task checks every `min(idle_timeout, 10s)` and removes entries whose `last_activity` exceeds the timeout:

```rust
// Use the stream normally
outbound.write_all(b"hello").await.unwrap();
let mut buf = vec![0u8; 1024];
let n = inbound.read(&mut buf).await.unwrap();

// Now wait for the idle timeout
tokio::time::sleep(IDLE_TIMEOUT + Duration::from_secs(2)).await;
```

## Observing EOF

After the router cleans up the stream, the inbound receiver's channel is closed. A `read()` returns 0 bytes (EOF):

```rust
let n = inbound.read(&mut buf).await.expect("read failed");
assert_eq!(n, 0); // EOF — stream cleaned up
```

## Running it

```bash
cargo run --example stream_idle_timeout
```
