# Throughput

import { Callout } from 'nextra/components'

> Source: [`examples/stream_throughput.rs`](https://github.com/nymtech/nym/blob/develop/sdk/rust/nym-sdk/examples/stream_throughput.rs)

This example sends 1 MB of random data over a single `MixnetStream` and verifies the receiver gets an identical copy. It tests sustained throughput and data integrity through the Mixnet.

## What this example shows

- Transferring a large payload over a stream
- Using `write_all` and `read_exact` for reliable transfer
- Verifying data integrity after transit through the Mixnet
- Concurrent send and receive with `tokio::join!`

## Setup

Two clients: a sender and a receiver. The receiver sets up a listener and the sender opens a stream:

```rust
let mut sender = mixnet::MixnetClient::connect_new().await.unwrap();
let mut receiver = mixnet::MixnetClient::connect_new().await.unwrap();
let recv_addr = *receiver.nym_address();

let mut listener = receiver.listener().unwrap();
let mut tx = sender.open_stream(recv_addr, None).await.unwrap();
let mut rx = listener.accept().await.unwrap();
```

## Sending 1 MB

The sender writes the entire payload in one `write_all` call. Under the hood, this is split into multiple Sphinx packets by the Mixnet:

```rust
let mut payload = vec![0u8; 1024 * 1024]; // 1 MB
rand::rngs::OsRng.fill_bytes(&mut payload);

tx.write_all(&payload).await.unwrap();
tx.flush().await.unwrap();
```

## Receiving

The receiver uses `read_exact` to read exactly 1 MB. This is important because the stream has no close/EOF semantics yet — the receiver needs to know the expected size:

```rust
let mut buf = vec![0u8; SIZE];
rx.read_exact(&mut buf).await.unwrap();
assert_eq!(buf, payload);
```

<Callout type="warning">
**No message ordering.** The Mixnet does not guarantee packet ordering. For this example, `read_exact` will block until all bytes arrive, but they may arrive out-of-order internally. For small payloads this works because each write fits in a single Sphinx packet. For large payloads, you may need a reordering layer (like the one in `tcp_proxy`) if byte-level ordering matters.
</Callout>

## Running it

This example has a 300-second timeout since it needs to send 1 MB through the Mixnet:

```bash
cargo run --example stream_throughput
```
