# Tutorial: Send Your First Private Message

import { Callout } from 'nextra/components'

In this tutorial you'll build a small Rust program that connects to the Nym Mixnet, sends a message to itself, receives it, and then replies anonymously using SURBs (Single Use Reply Blocks). By the end you'll understand the core messaging workflow and how anonymous replies work.

## What you'll learn

- Connecting an ephemeral client to the Mixnet
- Sending a message through Sphinx packet encryption
- Receiving and filtering incoming messages
- Replying anonymously via SURBs (without knowing the sender's address)
- Persisting client identity across restarts

## Prerequisites

- Rust toolchain (1.70+)
- A working internet connection (the client connects to the live Nym Mixnet)

## Step 1: Set up the project

Create a new Rust project and add the dependencies:

```sh
cargo init nym-hello
cd nym-hello
```

Add `nym-sdk` and `tokio` to your `Cargo.toml`:

```toml
[dependencies]
nym-sdk = "1.20.4"
tokio = { version = "1", features = ["full"] }
```

See [Importing](../importing) for alternative ways to import the crate (e.g. from Git for unreleased changes).

## Step 2: Connect and send a message

Replace the contents of `src/main.rs`:

```rust
use nym_sdk::mixnet::{self, MixnetMessageSender};

#[tokio::main]
async fn main() {
    // Connect an ephemeral client — keys are generated in memory
    // and discarded on disconnect.
    let mut client = mixnet::MixnetClient::connect_new().await.unwrap();

    let our_address = client.nym_address();
    println!("Connected! Our address: {our_address}");

    // Send a message to ourselves. The message travels through
    // 5 mix nodes (3 layers + gateway on each end) before arriving.
    client
        .send_plain_message(*our_address, "hello from the mixnet!")
        .await
        .unwrap();

    println!("Message sent — waiting for it to arrive...");
```

`send_plain_message` is async, but it **only blocks until the message is queued** — the actual Sphinx encryption, routing, and mixing happen in background tasks. The client must stay alive long enough for the message to traverse the Mixnet (typically a few seconds).

## Step 3: Receive the message

Still in `main()`, add:

```rust
    // Wait for incoming messages. Filter out empty ones — these
    // are SURB replenishment requests, not real data.
    let message = loop {
        if let Some(msgs) = client.wait_for_messages().await {
            if let Some(msg) = msgs.into_iter().find(|m| !m.message.is_empty()) {
                break msg;
            }
        }
    };

    let text = String::from_utf8_lossy(&message.message);
    println!("Received: {text}");
```

<Callout type="info">
**Why filter empty messages?** The SDK automatically manages reply SURBs. Occasionally the recipient requests more SURBs from the sender — these arrive as messages with an empty payload. In production code, always filter or handle them.
</Callout>

## Step 4: Reply anonymously with SURBs

Every message includes a `sender_tag` — an `AnonymousSenderTag` that lets the recipient reply **without knowing the sender's Nym address**. The reply travels back through pre-built SURB packets.

```rust
    // The sender_tag lets us reply without knowing who sent the message.
    let sender_tag = message
        .sender_tag
        .expect("message should include a sender tag");

    println!("Replying via SURBs (sender tag: {sender_tag})...");

    client
        .send_reply(sender_tag, "hello back, anonymously!")
        .await
        .unwrap();
```

## Step 5: Receive the reply and disconnect

```rust
    // Wait for the anonymous reply
    let reply = loop {
        if let Some(msgs) = client.wait_for_messages().await {
            if let Some(msg) = msgs.into_iter().find(|m| !m.message.is_empty()) {
                break msg;
            }
        }
    };

    println!("Reply: {}", String::from_utf8_lossy(&reply.message));

    // Always disconnect gracefully — the client writes to local
    // storage for SURB management and needs a clean shutdown.
    client.disconnect().await;
}
```

## Step 6: Run it

```sh
cargo run
```

You'll see output like:

```
Connected! Our address: 8gk4Y...@2xU4d...
Message sent — waiting for it to arrive...
Received: hello from the mixnet!
Replying via SURBs (sender tag: EfMGC...)...
Reply: hello back, anonymously!
```

The message took a round trip through the Mixnet — encrypted, mixed across multiple nodes, and delivered. The reply used SURBs, so the "replying" side never learned the original sender's address.

## Going further: persist your identity

So far we used `connect_new()` which creates ephemeral keys — discarded on disconnect. To keep the same Nym address across restarts, use the builder with on-disk storage:

```rust
use nym_sdk::mixnet::{MixnetClientBuilder, StoragePaths};
use std::path::PathBuf;

let storage = StoragePaths::new_from_dir(&PathBuf::from("/tmp/my-nym-client")).unwrap();

let client = MixnetClientBuilder::new_with_default_storage(storage)
    .await
    .unwrap()
    .build()
    .unwrap()
    .connect_to_mixnet()
    .await
    .unwrap();

// This address will be the same every time you run with the same path.
println!("Persistent address: {}", client.nym_address());
```

## Complete code

Here's the full `src/main.rs`:

```rust
use nym_sdk::mixnet::{self, MixnetMessageSender};

#[tokio::main]
async fn main() {
    let mut client = mixnet::MixnetClient::connect_new().await.unwrap();
    let our_address = client.nym_address();
    println!("Connected! Our address: {our_address}");

    // Send a message to ourselves
    client
        .send_plain_message(*our_address, "hello from the mixnet!")
        .await
        .unwrap();
    println!("Message sent — waiting for it to arrive...");

    // Receive (filtering empty SURB requests)
    let message = loop {
        if let Some(msgs) = client.wait_for_messages().await {
            if let Some(msg) = msgs.into_iter().find(|m| !m.message.is_empty()) {
                break msg;
            }
        }
    };
    println!("Received: {}", String::from_utf8_lossy(&message.message));

    // Reply anonymously via SURBs
    let sender_tag = message.sender_tag.expect("should have sender tag");
    client
        .send_reply(sender_tag, "hello back, anonymously!")
        .await
        .unwrap();

    // Receive the reply
    let reply = loop {
        if let Some(msgs) = client.wait_for_messages().await {
            if let Some(msg) = msgs.into_iter().find(|m| !m.message.is_empty()) {
                break msg;
            }
        }
    };
    println!("Reply: {}", String::from_utf8_lossy(&reply.message));

    client.disconnect().await;
}
```

## What you've learned

- **`MixnetClient::connect_new()`** creates an ephemeral client with auto-configured gateway and keys
- **`send_plain_message()`** queues a message for Sphinx encryption and mixing
- **`wait_for_messages()`** returns batches of reconstructed messages (filter empty ones)
- **`sender_tag`** + **`send_reply()`** enable anonymous replies without knowing the sender's address
- **`MixnetClientBuilder::new_with_default_storage()`** persists identity to disk
- **Always call `disconnect()`** for clean shutdown
