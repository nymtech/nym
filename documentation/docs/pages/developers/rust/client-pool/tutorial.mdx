# Tutorial: Handle Bursty Traffic with Client Pool

import { Callout } from 'nextra/components'

Use `ClientPool` to handle bursts of concurrent Mixnet operations without blocking on client creation.

<Callout type="info">
This tutorial assumes you've already added `nym-sdk` and `tokio` to your project. See [Installation](../importing) if not.
</Callout>

## Create and start the pool

The pool maintains a **reserve** of connected clients. The `start()` background loop creates replacements whenever the count drops below the target.

```rust
use nym_sdk::client_pool::ClientPool;
use nym_sdk::mixnet::MixnetMessageSender;
use std::time::Duration;

#[tokio::main]
async fn main() {
    let pool = ClientPool::new(3);
    let pool_bg = pool.clone();
    tokio::spawn(async move { pool_bg.start().await.unwrap() });

    // Wait for clients to connect (each takes several seconds)
    tokio::time::sleep(Duration::from_secs(15)).await;
    println!("Pool has {} clients", pool.get_client_count().await);
```

## Pop clients under load

`get_mixnet_client()` removes a client from the pool and returns it. The pool creates a replacement in the background.

```rust
    let mut handles = vec![];
    for i in 1..=3 {
        let pool = pool.clone();
        handles.push(tokio::spawn(async move {
            let client = match pool.get_mixnet_client().await {
                Some(c) => c,
                None => {
                    // Pool empty — create one on the fly (slower)
                    nym_sdk::mixnet::MixnetClient::connect_new().await.unwrap()
                }
            };

            let addr = *client.nym_address();
            client.send_plain_message(addr, format!("task {i}")).await.unwrap();

            if let Some(msgs) = client.wait_for_messages().await {
                for msg in msgs.iter().filter(|m| !m.message.is_empty()) {
                    println!("Task {i}: {}", String::from_utf8_lossy(&msg.message));
                }
            }

            client.disconnect().await;
        }));
    }

    for h in handles { h.await.unwrap() }
```

## Shut down

```rust
    // Wait for replenishment, then shut down
    tokio::time::sleep(Duration::from_secs(15)).await;
    println!("Pool has {} clients", pool.get_client_count().await);

    pool.disconnect_pool().await;
}
```

## Run it

```sh
cargo run
```

```
Pool has 3 clients
Task 1: task 1
Task 2: task 2
Task 3: task 3
Pool has 3 clients
```

Clients are **consumed, not returned** — the pool creates replacements automatically. If your application only needs one client at a time, use `MixnetClient::connect_new()` directly.
