# Cover Traffic

Cover traffic is a fundamental mechanism that provides **unobservability**вҖ”the property that observers cannot determine when real communication is occurring.

## The Problem: Traffic Analysis

Even with perfect encryption and mixing, an adversary can learn information from:

- **Volume**: How much data is being sent
- **Timing**: When data is being sent
- **Patterns**: Regular schedules, bursts of activity

Over time, these patterns can:
- Identify regular communication partners
- Reveal work schedules and habits
- Correlate activity across services
- Enable long-term statistical deanonymization

## The Solution: Constant Traffic

> The Nym mixnet generates cover traffic "loops" ... Loops generated by mix nodes ensure a minimum level of anonymity at all times, while end users can generate loops to obfuscate the timing and volume of their active communication through Nym and thus achieve unobservability.
>
> вҖ” [Nym Whitepaper](https://nym.com/nym-whitepaper.pdf) В§4

## How It Works

### Client-Side Cover Traffic

Connected clients continuously send packets at a rate determined by a Poisson process:

1. When real data needs to be sent, it replaces a cover packet
2. When no real data is pending, cover (dummy) packets are sent
3. The rate remains constant regardless of actual activity

This serves multiple purposes:
- Tests network routes are functioning
- Provides traffic for mixing with others' cover traffic
- Detects potential active attacks
- Maintains anonymity set even during idle periods

### Node-Generated Cover Traffic

Mix nodes also generate their own cover traffic:
- Ensures minimum traffic level even with few users
- Provides baseline anonymity guarantees
- Prevents traffic analysis of individual nodes

## Properties Achieved

### Unobservability

External observers cannot determine:
- Whether a user is actively communicating
- How much real data is being transmitted
- When communication sessions begin or end

### Resistance to Statistical Attacks

Long-term observation yields no useful information:
- Traffic patterns are constant
- No correlation between user activity and traffic volume
- Behavioral patterns are hidden

### Minimum Anonymity Guarantee

Even with few active users, cover traffic ensures:
- Mix nodes always have traffic to mix
- Anonymity sets remain meaningful
- Individual users don't stand out

## Poisson Process

Traffic generation follows a Poisson process with parameter О» (lambda):

- Inter-packet times are exponentially distributed
- Mean rate of О» packets per second
- Memoryless: each packet independent of previous
- Provides optimal mixing properties

The Poisson distribution was chosen because:
- It models continuous-time random events
- It provides maximum entropy for a given mean rate
- It's computationally efficient to generate

## Configuration

Cover traffic parameters are configurable:
- Higher rates provide better unobservability but use more bandwidth
- Lower rates reduce resource usage but may allow some inference
- Default parameters balance privacy and performance

## Trade-offs

| Aspect | More Cover Traffic | Less Cover Traffic |
|--------|-------------------|-------------------|
| Privacy | Better unobservability | More traffic visible |
| Bandwidth | Higher usage | Lower usage |
| Latency | Minimal impact | Minimal impact |
| Cost | More credential usage | Less credential usage |
